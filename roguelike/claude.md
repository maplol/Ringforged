# Ringforged — Roguelike по вселенной «Повелитель Колец»

## Обзор проекта

Бесконечный top-down roguelike шутер на React + Canvas 2D. Привязан к лору ранобэ «Ringforged» (мир Ауракс). Игрок управляет персонажем, который собирает магические кольца стихий, комбинирует их для получения мощных эффектов и сражается с фракциями мира Ауракс через процедурно сгенерированные подземелья.

После прохождения 8 стадий (по томам ранобэ) начинается новый цикл с усиленными врагами. Каждый цикл добавляет +2 комнаты к каждой стадии и масштабирует HP/урон врагов.

**Язык интерфейса**: полностью русский.

## Архитектура

### Двойной стейт

Проект использует гибридный подход к состоянию:

- **`stateRef` (mutable ref)** — основной источник правды для игрового цикла. Содержит `GameState` с позициями, HP, списками врагов, снарядов, частиц. Мутируется напрямую в `requestAnimationFrame` без React-перерендеров.
- **`useState` (React state)** — зеркало для HUD-элементов. Обновляется каждые ~4 кадра (условие `Math.floor(gameTime * 60) % 4 === 0`) переливанием данных из `stateRef`.

Это даёт 60 FPS рендеринг через canvas без React-overhead, но ценой дублирования.

### Game loop

Один `useEffect` с зависимостью `[stageIndex]` содержит `requestAnimationFrame`-цикл:

```
requestAnimationFrame → dt (ограничен 50мс) →
  1. Обработка ввода (WASD + мышь)
  2. Движение / рывок (Shift)
  3. Регенерация стамины
  4. Обнаружение активного комбо
  5. Стрельба (ЛКМ/ПКМ + авто-ауры)
  6. AI врагов (chase/shoot/dash/boss)
  7. Движение снарядов
  8. Коллизии (снаряд→враг, снаряд→игрок, контакт)
  9. Коллизии с препятствиями
  10. Сбор XP-орбов (магнит)
  11. Обновление частиц, пылинок, ambient-эффектов
  12. Очистка мёртвых сущностей
  13. Проверка level-up → экран улучшения
  14. Проверка очистки комнаты → открытие дверей
  15. Проверка портала выхода
  16. Проверка перехода через дверь
  17. Обновление камеры (zoom, lerp, shake)
  18. HUD sync (каждые ~4 кадра)
  19. renderFrame()
```

### Кольцо (Ring)

```typescript
interface Ring {
  id: string;
  element: ElementType;
  secondElement?: ElementType;   // Только у слитых комбо-колец
  comboName?: string;            // Название комбо после слияния
  rank: RingRank;                // Lower → Middle → Higher → Legendary
  name: string;
  level: number;                 // 1–5
  attackType: "projectile" | "aura" | "melee";
  slotType: SlotType;
}
```

### Ранги колец

| Уровень | Ранг |
|---------|------|
| 1–2 | Lower (Низший) |
| 3 | Middle (Средний) |
| 4 | Higher (Высший) |
| 5 | Legendary (Легендарный) |

Пассивный бонус для стихии, привязанной к ядру:

| Элемент | Бонус | Эффект |
|---------|-------|--------|
| Огонь | `explosive_chance` | 15% шанс взрыва AoE при попадании |
| Вода | `slow_enhance` | +40% длительность замедления |
| Ветер | `proj_speed` | +30% скорость снарядов |

## Комбо-система (14 комбинаций)

Когда у игрока есть 2+ кольца разных элементов — автоматически активируется комбо-эффект. Приоритет: слитые кольца (с `secondElement`) > пары из разных слотов.

| Комбо | Элементы | Тип | Эффект |
|-------|----------|-----|--------|
| Шторм | Ветер + Вода | `aoe_radius` | +40% радиус атак |
| Огненный Шторм | Ветер + Огонь | `dot` | 8 ед/сек поджог |
| Метель | Вода + Лёд | `slow` | 40% замедление |
| Проводимость | Молния + Вода | `chain` | Молния на 2 доп. цели |
| Лава | Земля + Огонь | `dot` | 12 ед/сек поджог |
| Чёрная Дыра | Гравитация + Тьма | `pull` | Притяжение врагов (400) |
| Фотосинтез | Свет + Природа | `regen` | +5 HP/сек |
| Плазма | Огонь + Молния | `explosive` | AoE-взрыв 60 ед. |
| Метеор | Земля + Гравитация | `pierce` | Пробивание насквозь |
| Бездонный Мороз | Тьма + Лёд | `lifesteal` | 15% вампиризм |
| Священная Молния | Свет + Молния | `stun` | Оглушение 0.5с |
| Небесный Ход | Ветер + Гравитация | `aoe_radius` | +30% скорость |
| Бритвенный Лист | Природа + Ветер | `dot` | 6 ед/сек поджог |
| Цветение | Вода + Природа | `regen` | +3 HP/сек |

### Слияние колец

При получении нового кольца, если одно из имеющихся образует с ним комбо, игроку предлагается **слить** их. Слияние:
- Два кольца физически объединяются в **одно** кольцо с `secondElement`
- Занимает **один** слот (слот партнёра)
- Освобождает слот, который мог бы занять новое кольцо → слот не тратится
- Уже слитые кольца не предлагаются для повторного слияния
- `findPossibleCombo` пропускает кольца с `comboName`/`secondElement` и core-слот

```

DMG = baseDamage × (1 + level × 0.18)
```

## Снаряды (`Projectile`)

```typescript
interface Projectile {
  id: string;
  x, y, vx, vy: number;
  owner: "player" | "enemy";
  damage: number;
  element?: ElementType;
  hp: number;              // При 0 — удаляется
  lifeTime: number;        // Время жизни в секундах
  radius: number;          // Радиус коллизии
  piercing: boolean;       // Пробивание (от комбо Метеор)
  chainCount: number;      // Текущее число цепных прыжков
}
```

### Формулы урона снаряда игрока

```
baseDmg = (PROJECTILE_BASE_DAMAGE + ring.level × PROJECTILE_DAMAGE_PER_LEVEL) × player.damageMult
finalDmg = applyCoreDamage(baseDmg, element, player)
speed = PROJECTILE_BASE_SPEED × player.projectileSpeedMult
cooldown = PLAYER_BASE_ATTACK_COOLDOWN / player.attackSpeed
```

### Ауры (слоты 2–3)

Автоматически стреляют AoE-снарядом вокруг игрока:
- Слот 2: кулдаун 1.5с / attackSpeed
- Слот 3: кулдаун 1.8с / attackSpeed
- Урон: `(10 + ringLevel × 5) × damageMult × 0.6`
- Радиус: `55 + ringLevel × 10`
- Piercing: true

## Карта и комнаты

### Процедурная генерация (`mapgen.ts`)

Карта — граф комнат на сетке `(x, y)`, хранится как `Record<string, Room>` с ключами `"x,y"`.

Алгоритм:
1. Стартовая комната в `(0, 0)`, cleared + visited
2. Frontier-based expansion: случайный рост из фронта
3. Boss-комната — последняя размещённая
4. Exit-комната — приклеивается к boss-комнате (случайное свободное направление)
5. Каждой комнате назначается тип: `start`, `normal`, `elite` (20%), `treasure` (12%), `boss`, `exit`

### Типы комнат

| Тип | Размеры (мин–макс) | Особенности |
|-----|-------------------|-------------|
| `start` | 1000×750 (фикс.) | Без врагов, cleared |
| `normal` | 1200–1600 × 900–1200 | 3+ врагов, препятствия |
| `elite` | 1600–2000 × 1200–1500 | 4+ усиленных врагов (level+1), препятствия |
| `treasure` | 900×700 (фикс.) | Без врагов, +30 HP |
| `boss` | 2000–2400 × 1500–1800 | Босс + 2+ охранников, препятствия |
| `exit` | 800×600 (фикс.) | Портал для перехода на следующую стадию |

### Двери

Каждая комната хранит `doors: { top, bottom, left, right }` (boolean). Двери открываются после очистки комнаты.

Переход: игрок подходит к стене в зоне двери → `checkDoor` → `enterRoom` → спавн врагов в новой комнате.

### Препятствия (`Obstacle`)

| Тип | Радиус | HP | Поведение |
|-----|--------|-----|----------|
| `pillar` | 18–30 | 999 | Неразрушимый столб, блокирует снаряды и движение |
| `barrel` | 14–20 | 2 | Разрушается, взрывается в радиусе 80px (30 урона) |
| `lava` | 30–55 | 999 | Наносит 8 HP/сек при контакте (игроку и врагам) |

## Система улучшений

### Прокачка

- **XP**: орбы с врагов (10–19 XP за обычного, больше за элитных/босса)
- **Level-up**: XP >= maxXp → экран выбора улучшений
- **Формула maxXp**: начало 40, каждый уровень × 1.35

### Генерация улучшений (`generateUpgrades`)

При level-up генерируется 3 случайных улучшения (из 5 возможных):

1. **Новое кольцо** (`type: "ring"`) — если есть свободный слот
   - Элемент: случайный из ещё не имеющихся (учитывает `secondElement` слитых колец)
   - Если у игрока есть кольцо, образующее комбо с новым → в карточке появляется чекбокс «Слить»
2. **Стихийное ядро** (`type: "core"`) — только для 5-го слота, элемент из уже имеющихся
3. **Усиление кольца** (`type: "ringLevelUp"`) — +1 уровень (макс. 5), случайное не-core кольцо
4. **Стат** (`type: "stat"`) — по `statKey`:
   - `hp`: +25 макс. HP
   - `atk`: +20% урон
   - `spd`: +15% скорость атаки
   - `projSpd`: +20% скорость снарядов
   - `projCnt`: +1 снаряд
   - `staminaMax`: +30 макс. стамины

- Затухание: скорость × `PLAYER_DASH_DECAY` (0.88) каждый кадр

## Камера

- **Zoom**: фиксированный для обычных комнат (`Math.max(vw/1200, vh/900)`), уменьшенный для boss-комнат (× 0.7)
- **Следование**: `lerp` с коэффициентом 0.08 к позиции игрока
- **Границы**: `clamp` к краям комнаты, чтобы не видеть пустоту
- **Screen shake**: при убийствах. Boss kill = 0.4с, обычный = 0.1с. Интенсивность = `shakeTimer × 20` × случайное смещение.

## Рендерер (`renderer.ts`)

Полностью процедурная пиксельная графика на Canvas 2D API:

### Комнаты
- **Пол**: каменные тайлы 48×48 с затиркой, случайными трещинами, виньеткой
- **Стены**: кирпичная кладка (16×8), тени, блики
- **Двери**: открытые (зелёное свечение + стрелка) / закрытые (железные решётки)
- **Факелы**: анимированные с мерцанием пламени и световым ореолом
- **Декоративные углы**: орнамент в каждом углу
- **Атмосфера**: boss-комнаты (красный оверлей + skull-метки), treasure (золотой), elite (фиолетовый)
- **Портал** (exit-комнаты): вращающееся кольцо с частицами

### Персонажи
Все нарисованы процедурно (beginPath / lineTo / fill):
- **Игрок**: плащ с анимацией покачивания, руки с кольцами-свечениями, глаза, волосы, орбитальные кольца стихий
- **Chase-враг**: шлем, меч, анимация замаха
- **Shooter-враг**: капюшон, посох с навершием, анимация стрельбы
- **Dasher-враг**: рогатый шлем, большой меч, предупреждающее свечение перед рывком
- **Tank-враг**: квадратный щит, массивный корпус
- **Boss**: корона, плащ, расширенная модель

### Анимации
- **attackAnim**: отклонение руки/оружия на основе прогресса (0→1), светящиеся руки
- **hitFlash**: белая вспышка + лёгкий сдвиг позиции на 0.15с
- **isDashing** (игрок): шлейф полупрозрачных afterimages
- **invincibleTimer**: красная вспышка, пульсирующее кольцо щита

### Эффекты
- **Частицы**: стихийные искры при ударе, белая вспышка, угольки для боссов
- **Числа урона**: всплывают, масштабируются, обводка + тень для крупных
- **XP-орбы**: пульсация, парение, мягкое свечение
- **Ходьба**: пылевые частицы из-под ног
- **Ambient**: плавающие пылинки в комнате

### Препятствия
- **Столб**: 3D-колонна с тенью, текстурой, трещинами
- **Бочка**: деревянная с металлическими обручами, «!» предупреждение
- **Лава**: ломаные линии разломов с анимированным свечением
- **Кристалл**: фасеточный корпус, боковые осколки, свечение элемента

### Экраны (GameState.screen)

| Экран | Описание |
|-------|----------|

- **Верх**: `Том X/8 — Название стадии` + номер цикла + уровень игрока
- **Boss-бар**: красный градиент, имя + HP бар
- **Миникарта**: правый верхний угол, сетка комнат с цветовой кодировкой типов
- **Низ**: HP-бар (красный), Стамина-бар (бирюзовый) + dash-индикатор ⚡, XP-бар (жёлтый)

- **Комбо-бейдж**: название + описание активного комбо

### Экран улучшений (Hades-стиль)

- Вертикальные карточки с иконкой, названием, описанием, рамкой цвета редкости
- Два шага: выбор карточки → кнопка «Принять»
- Если возможно комбо-слияние: дополнительная секция с чекбоксом и описанием
- Защита от мисклика: кнопка неактивна пока не выбрана карточка

### Инвентарь (C / С)

Оверлей (не отдельный `screen`) с:
- Статы игрока (HP, стамина, урон, скорость атаки, скорость снарядов, снаряды, регенерация)
- Кольца в слотах с уровнями, рангами и элементами
- Стихийное ядро и его бонус
- Активное комбо и описание
- История всех применённых улучшений
- Активные баффы

## Известные архитектурные ограничения

1. **God Component**: `App.tsx` совмещает game loop, ввод, UI и state management в одном компоненте
2. **Двойной стейт**: mutable ref + React useState с периодической синхронизацией
3. **useEffect на [stageIndex]**: весь game loop пересоздаётся при смене стадии
4. **O(n×m) коллизии**: нет spatial partitioning, все снаряды проверяются против всех врагов
5. **shadowBlur**: ~39 использований в renderer.ts, дорогая операция Canvas
6. **CSS**: 800 строк кастомного CSS параллельно с Tailwind
7. **Процедурная графика**: каждый кадр заново рисуются все тайлы, факелы, стены (нет кэширования)
